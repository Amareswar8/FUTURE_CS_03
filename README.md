# FUTURE_CS_03
# API Security Risk Analysis (Modern SaaS Skill)

## 1. API Under Review

* **API Name:** FakeStoreAPI
* **URL:** https://fakestoreapi.com and https://fakestoreapi.com/docs
* **Purpose:** Demo e-commerce API for testing and learning
* **Environment:** Public, non-production, test API

## 2. Tools Used

* **Postman** – Sending and analyzing API requests and responses
* **MS Word / PDF** – Documentation and reporting

## 3. Methodology

1. Review public API documentation
2. Send permitted requests using API clients
3. Inspect responses for data exposure and errors
4. Review authentication and access control requirements
5. Map observations to common API security risks
6. Suggest remediation guidance
